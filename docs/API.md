# API

The backend is a local FastAPI service. The authoritative schema is the live OpenAPI document exposed at `/openapi.json`; frontend contracts are generated from it.

Base URL:

```text
http://127.0.0.1:8008
```

## Health

### `GET /health`

Returns service status, version, and environment.

```bash
curl http://127.0.0.1:8008/health
```

## Deterministic analysis

### `POST /api/v1/calculate`

Deterministic sample size, duration, assumptions, warnings, and optional `bonferroni_note`.

```bash
curl -X POST http://127.0.0.1:8008/api/v1/calculate ^
  -H "Content-Type: application/json" ^
  -d "{\"metric_type\":\"binary\",\"baseline_value\":0.042,\"mde_pct\":5,\"alpha\":0.05,\"power\":0.8,\"expected_daily_traffic\":12000,\"audience_share_in_test\":0.6,\"traffic_split\":[50,50],\"variants_count\":2}"
```

### `POST /api/v1/design`

Builds the deterministic report from the full experiment payload.

### `POST /api/v1/analyze`

Runs the full combined flow:

- deterministic calculation
- deterministic report generation
- optional orchestrator advice

```bash
curl -X POST http://127.0.0.1:8008/api/v1/analyze ^
  -H "Content-Type: application/json" ^
  -d @docs/demo/sample-project.json
```

### `POST /api/v1/llm/advice`

Optional advisory-only route for the local orchestrator.

## Project storage

### `GET /api/v1/projects`

List saved SQLite projects.

### `POST /api/v1/projects`

Create a new saved project.

### `GET /api/v1/projects/{project_id}`

Load a saved project payload.

### `PUT /api/v1/projects/{project_id}`

Update an existing project.

### `DELETE /api/v1/projects/{project_id}`

Delete a saved project.

```bash
curl -X DELETE http://127.0.0.1:8008/api/v1/projects/PROJECT_ID
```

## Project activity

### `GET /api/v1/projects/{project_id}/history`

Returns paginated analysis run and export event history.

Query params:

- `analysis_limit`
- `analysis_offset`
- `export_limit`
- `export_offset`

```bash
curl "http://127.0.0.1:8008/api/v1/projects/PROJECT_ID/history?analysis_limit=5&export_limit=5"
```

### `POST /api/v1/projects/{project_id}/analysis`

Persist the latest combined analysis snapshot for a saved project.

### `POST /api/v1/projects/{project_id}/exports`

Record a markdown/html export event, optionally linked to an analysis run id.

## Comparison

### `GET /api/v1/projects/compare`

Compare two saved projects by their latest or explicitly selected analysis runs.

Query params:

- `base_id`
- `candidate_id`
- `base_run_id` optional
- `candidate_run_id` optional

```bash
curl "http://127.0.0.1:8008/api/v1/projects/compare?base_id=BASE&candidate_id=CANDIDATE"
```

## Report export

### `POST /api/v1/export/markdown`
### `POST /api/v1/export/html`

Convert a deterministic report payload into exportable markdown or html.

## Validation notes

- supported variant count is `2..10`
- binary baselines must be between `0` and `1`
- continuous metrics require positive `baseline_value` and `std_dev`
- `traffic_split` length must match `variants_count`
- malformed request bodies return `422`
- domain errors return structured `400`
