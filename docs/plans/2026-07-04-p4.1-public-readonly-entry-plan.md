# P4.1 — Публичный read-only вход (гостевой лендинг + анонимный read-scope)

Аудит: `audit_fable_02_07_2026.md` §6.2 п.1 (публичный `/` не продаёт продукт) + п.4 (любой
посетитель может удалить демо). Концепт согласован с Юлей 04.07.2026 ДО кода:

1. **Гость = полный калькулятор.** Stateless compute-POSTы разрешены анониму; мутации
   состояния — только с write-токеном.
2. **LLM гостю — только BYO-ключ.** Серверный Mistral-ключ (insurance) работает только для
   write/admin-сессий; гость без `X-AB-LLM-Token` получает нейтральный отказ с кодом.
3. **Демо смотрятся в том же workspace UI** (визард + результаты + live-блок), write-кнопки
   скрыты; отдельной read-only страницы нет.

Деплой на Space (секреты `AB_API_TOKEN` + `AB_PUBLIC_DEMO=true`) — НЕ здесь: отдельный гейт
0.1 по явному «задеплой».

## Цель

Гость на `/` видит лендинг-обзор (4 демо-карточки + live-цифры + CTA), может пройти весь
калькулятор, но не может изменить ни один сохранённый объект — и это обеспечено сервером,
а не скрытием кнопок.

## Backend

- [ ] **B1. Settings:** `public_demo: bool` ← `AB_PUBLIC_DEMO` (default false) в `config.py`.
      Verify: `test_config.py` — парсинг/дефолт.
- [ ] **B2. Анонимный read-scope:** в `http_runtime.py` middleware — при `public_demo=true`,
      `auth_required` и отсутствии токена НЕ 401, а `auth_scope="read"`,
      `auth_source="anonymous"`, `audit_actor="anonymous"`. `public_demo=false` → поведение
      байт-в-байт прежнее.
- [ ] **B3. Compute-whitelist:** `PUBLIC_COMPUTE_PATHS` (frozenset) в `http_utils.py`:
      `/calculate`, `/sensitivity`, `/srm-check`, `/multiple-testing`, `/results`,
      `/results/categorical|paired|omnibus`, `/simulate/bandit`, `/assignment/preview`,
      `/design`, `/analyze`, `/llm/advice`, `/hypotheses/generate`, `/export/*` (4),
      `/projects/compare` (POST-вариант). Проверка «read-scope + не-GET → 403» пропускает
      эти пути. Route-гварды этих эндпоинтов: `require_write_auth` → `require_auth`.
      Семантика read-scope теперь = «ничего не мутирует» (распространяется и на
      `AB_READONLY_API_TOKEN`-сессии — задокументировать в docstring).
      Мутации НЕ трогаем: projects save/PUT/DELETE/archive/restore/analysis/exports,
      templates POST/use/DELETE, execution ingest-POSTы, `/experiments/{id}/assign`,
      workspace import, keys/webhooks (admin).
- [ ] **B4. LLM-гейт гостя:** в `analyze`/`llm_advice`/`hypotheses_generate` — server-side
      insurance (Mistral) и local-orchestrator-путь только при scope write/admin ЛИБО когда
      auth вообще выключен; read-scope без BYO-заголовков → advice `available=false` с
      `error_code="public_demo_llm_requires_key"` (не HTTP-ошибка — как seeded-путь 0.3).
      BYO (`X-AB-LLM-Provider`+`X-AB-LLM-Token`) работает для любого scope.
      i18n backend ×7 не нужен, если код идёт через существующий фронтовый паттерн
      (`AiAdviceSection` ветвится на error_code — фронтовые ключи ×7).
- [ ] **B5. Diagnostics:** `public_demo: bool` в `DiagnosticsAuthSummary` (contract regen).
- [ ] **B6. Тесты backend:** новый `test_public_demo.py` — анон GET projects 200; анон POST
      `/calculate`+`/analyze`+`/results` 200; анон DELETE/POST projects 403; анон templates
      POST 403; keys 401; `/analyze` анонима НЕ зовёт Mistral (monkeypatch-адаптер, assert
      not called) и отдаёт `public_demo_llm_requires_key`; `/analyze` анонима с BYO-заголовком
      зовёт openai-адаптер; write-токен в public_demo мутирует как раньше;
      `public_demo=false` → анон 401 (регрессия). Read-token-сессия теперь может POST
      `/calculate` (осознанное изменение семантики — явный тест).
