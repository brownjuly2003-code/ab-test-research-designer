# P3.1 — Paired-family analyzers (paired t + Wilcoxon signed-rank + McNemar)

Audit finding §4 Tier 1 п.1: paired within-subject designs (pre/post on the same units) had
**zero** coverage. This slice adds the second-most-common test class after independent two-sample,
as a new standalone "paired samples" section — one raw-input form feeding three analyzers,
mirroring the Categorical (chi-square) precedent.

## Design decisions

- **New standalone section**, not a toggle inside `ObservedResultsSection`. The existing toggle
  offers alternative tests *on the same summary/ranked data for a given base metric type*; paired
  tests need a fundamentally different input (two **equal-length** arrays, paired by index), so they
  get their own request/response/endpoint/frontend section — exactly as Categorical did.
- **One input form for all three** ("третья форма «paired values»" from the audit): `control_values`
  + `treatment_values` of equal length. `test_type ∈ {paired_t, wilcoxon, mcnemar}`.
  McNemar additionally requires every value ∈ {0, 1} (paired binary).
- **Dedicated `PairedResultsResponse`** (like `CategoricalResultsResponse`): a scalar effect + CI is
  natural for paired_t (mean diff) and Wilcoxon (Hodges–Lehmann pseudomedian), and McNemar reports a
  marginal proportion difference; paired-specific fields (n_zero_differences, discordant b/c) are
  optional, as `ResultsResponse` already does for effect_size.
- **stdlib-only, pure functions** in `app/stats/paired.py`; response assembled in the service layer.
  Reuses `student_t.t_cdf`/`t_ppf`, `binary.normal_ppf`, `srm.chi_square_cdf` — no new special fns.

## Statistics (frozen against scipy 1.17.1 / statsmodels 0.14.6 BEFORE coding)

Verification script `scratchpad/verify_paired_vs_scipy.py`. Differences defined `d_i = treatment_i − control_i`.

- **paired t** = one-sample t on d: `t = mean(d)/(sd(d,ddof=1)/√n)`, df=n−1, two-sided p via `t_cdf`.
  CI = mean(d) ± t_{1−α/2,df}·se. Effect size Cohen's d_z = mean(d)/sd(d).
  Frozen T1 (n=10): t=5.6667, p=0.000307, mean_d=1.7, CI=(1.0214, 2.3786), d_z=1.7920.
- **Wilcoxon signed-rank** = drop zero diffs, rank |d| with **midranks on ties**, W+/W− rank sums.
  Normal approx with **continuity correction + tie correction**: T=min(W+,W−),
  μ=n_r(n_r+1)/4, σ²=n_r(n_r+1)(2n_r+1)/24 − Σ(t³−t)/48, z=(T−μ+0.5)/σ, p=2·Φ(−|z|).
  Matches `scipy.stats.wilcoxon(mode="approx", correction=True, zero_method="wilcox")`.
  Effect size rank-biserial r=(W+−W−)/(n_r(n_r+1)/2). Location = Hodges–Lehmann pseudomedian
  (median of Walsh averages (d_i+d_j)/2, i≤j) + distribution-free CI (same normal-approx rank
  offset as the two-sample MW `_hodges_lehmann_shift`).
  Frozen T2 (2 zeros, ties |d|=2×6): p=0.015673, r=0.9444, HL=+2.0, CI=(0.5,2.0).
  Frozen T3 (1 zero): p=0.008258, r=1.0, HL=+1.5, CI=(1.0,2.5).
- **McNemar** (paired binary) = discordant counts b (0→1), c (1→0). n_disc=b+c.
  Exact two-sided binomial (p=0.5) when n_disc < 25 (`MCNEMAR_EXACT_MAX_DISCORDANT`), else
  Edwards continuity-corrected chi-square `(|b−c|−1)²/(b+c)`, df=1, p via `chi_square_cdf`.
  Matches `statsmodels ... mcnemar`. Effect = marginal proportion difference (b−c)/n_pairs;
  effect size = odds ratio b/c (None when c=0). n_disc=0 → p=1, not significant.
  Frozen T4 (b=7,c=0): exact p=0.015625. T5 (b=39,c=18): chi²=7.0175, p=0.008071. T6 (b=c=0): p=1.
  Edge verified vs statsmodels: b==c=30 chi²=0.01667/p=0.897; b=3,c=1 exact p=0.625.
