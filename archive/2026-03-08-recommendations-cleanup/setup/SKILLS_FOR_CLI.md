# Skills / instruction packs for token efficiency

Ты планируешь использовать CLI, поэтому полезнее не огромные skills, а маленькие повторно используемые instruction packs.

## Главный принцип
Для экономии токенов лучше иметь 4-6 коротких skills по ролям, чем один монструозный skill на всё.

## Рекомендуемый набор

### 1. `repo-study`
Назначение:
- сначала изучить репозиторий,
- понять структуру,
- найти точки интеграции,
- не писать код до короткого плана.

Что внутри должно быть:
- прочитай docs/
- опиши текущую архитектуру
- перечисли риски
- предложи минимальный план
- не меняй файлы до подтвержденного плана или until explicit request to implement

### 2. `phased-implementation`
Назначение:
- реализовывать только текущую фазу,
- не убегать вперед.

Что внутри:
- работай только по текущей фазе
- не трогай unrelated files
- после завершения обнови progress.md
- результат должен быть runnable

### 3. `deterministic-math-only`
Назначение:
- для stats engine.

Что внутри:
- никакой LLM-математики
- только deterministic formulas and code
- добавляй unit tests
- явно документируй assumptions

### 4. `safe-refactor`
Назначение:
- безопасные изменения без разрастания контекста.

Что внутри:
- сначала опиши, что меняешь
- меняй минимальный набор файлов
- сохраняй API contracts
- не делай широких архитектурных перестроек

### 5. `llm-adapter-integration`
Назначение:
- работа с локальным оркестратором.

Что внутри:
- сначала изучи `D:\Perplexity_Orchestrator2`
- найди реальный способ вызова: process / script / local HTTP / config
- не хардкодь путь и secrets
- сделай graceful fallback

### 6. `beginner-friendly-output`
Назначение:
- чтобы ответы Codex были понятны тебе.

Что внутри:
- объясняй кратко, человеческим языком
- отдельно пиши: что сделал, что осталось, как проверить
- не сыпь лишним жаргоном

## Какой размер skill оптимален
Ориентир:
- 15-40 строк на skill
- без длинных примеров
- без огромных справочников

## Что невыгодно
Плохо для токенов:
- один гигантский skill на 500+ строк
- дублирование одних и тех же правил в каждом skill
- огромные style guides для кода на ранней стадии проекта

## Минимальный лучший набор
Если хочешь совсем экономно, начни с 3 skills:
- `repo-study`
- `phased-implementation`
- `llm-adapter-integration`

Этого уже хватит для старта.
